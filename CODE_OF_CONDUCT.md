# Code of Conduct

## Our standard

BitsFactor Scripts should remain a practical, respectful, low-drama project.

People participating in the project are expected to:
- be respectful and constructive
- focus on the code, behavior, and user impact
- assume good intent before escalating
- give actionable feedback instead of dismissive feedback
- avoid personal attacks, harassment, and inflammatory behavior

## Unacceptable behavior

Examples of unacceptable behavior include:
- harassment or intimidation
- insults, slurs, or demeaning language
- repeated bad-faith disruption
- publishing someone else's private information without permission
- posting secrets or sensitive access material in issues or PRs

## Project-specific expectations

Because this repository includes bootstrap and credential-adjacent tooling:
- never post API keys or private SSH keys
- avoid sharing production host details unless necessary
- keep security reports minimally public until the maintainer has reviewed them

## Enforcement

Maintainers may remove comments, close discussions, or limit participation when behavior makes the project unsafe or unproductive.

## Reporting conduct issues

If you run into a conduct issue, contact the maintainer through the most direct channel you already have. If that is not possible, open a minimal issue requesting follow-up without posting sensitive or inflammatory detail publicly.

## Scope

This code of conduct applies to repository discussions, issues, pull requests, and review interactions related to this project.
